# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** lipgloss ends every rendered styled segment with a full SGR reset, which drops any background inherited from an enclosing style — so composing multiple styled segments (chips, separators, markers) inside a bg-carrying column style leaves only the FIRST segment on the intended bg; the rest fall to terminal-default (SGR `49m`). Fix: each segment AND separator must self-carry its background explicitly (`lipgloss.NewStyle().Background(bg).Render(...)`), and thread the row background through so selected/highlighted rows keep their selection bg.
- **[gotcha]** `.golangci.yml` sets `run: tests: false`, so the `unused` linter does NOT count `_test.go` references — a production function called ONLY from tests is flagged `func (*App).X is unused` and `make check`/`make check-test` fails. This is the standard TDD trap: helpers written before they're wired into production code. Suppress with a `//nolint:unused // used in tests` directive placed on its own line directly above the func (or after the doc comment, separated by a blank `//` line), matching existing precedent in `layer_helpers.go`, `overlay_labels.go`, and `config.go`; run `gofmt -w` after adding it.
- **[correction]** Label rendering for the tree labels column and detail pane must NOT use `renderPillChip` — its nerd-font pill caps (`ue0b6`/`ue0b4`) render as tofu (□?) in terminals lacking a patched font. Use `renderLabelTag(label, hex)`: plain text on the theme background when no custom color is set, a padded colored block only when a custom color is configured. Verify by byte-counting the glyphs in a tmux capture.
- **[gotcha]** A render-comparison UI test that asserts two styled rows/segments DIFFER by background (or checks for specific `48;2;R;G;B` bg bytes) FALSELY FAILS under the default test color profile — with no profile set, lipgloss drops backgrounds so a "selected" row renders byte-identical to its unselected baseline. Set `lipgloss.SetColorProfile(termenv.TrueColor)` at the top of the test (save/restore: `prev := lipgloss.ColorProfile()` + `defer lipgloss.SetColorProfile(prev)`) so background differences actually appear in the output. Seen: `TestBuildTreeLines_MultiSelectRowBackground` failed with "expected selected non-cursor row to render differently from its unselected baseline" until TrueColor was set.
- **[gotcha]** A test that calls `lipgloss.SetColorProfile(...)` mutates GLOBAL lipgloss state and leaks into other tests in the package — a color-dependent assertion elsewhere then fails ONLY in the full `go test ./internal/ui` run, not when the test is run in isolation. Always save and restore: `prev := lipgloss.ColorProfile(); lipgloss.SetColorProfile(termenv.TrueColor); defer lipgloss.SetColorProfile(prev)`. (The symbol is `termenv.TrueColor`, not `lipgloss.TrueColor`.)
- **[gotcha]** `./scripts/tui-test.sh` launches `abacus` against the REAL `./.beads` DB, so any action/write keystroke MUTATES live beads — a prior smoke test closed real bead `ab-c3j` by sending keys. Restrict smoke tests to nav/view keys ONLY (`j`/`k`/`J`/`K`/`l`/`h`/`v`/`o`/quit); NEVER send the mutation keys `s`/`p`/`L`/`c`/`m`/`N`/`delete`/`enter` unless you intend to edit production data.
- **[pattern]** A new persistent tree-view dimension (filter, view mode, toggle) must be threaded through FOUR places or it silently breaks: the `App` struct (`app.go`) AND the `viewState` struct (`state.go`), `captureState()` (`state.go`), and the restore block in `refresh.go` (next to `m.viewMode = state.viewMode`). Adding it to only `viewState` fails the build; skipping the `refresh.go` restore makes the value vanish on the ~10s auto-refresh. Derived, index-based state (like a multi-select anchor) is instead reset in `recalcVisibleRows` since row indices shift on refresh/filter/view-mode.
- **[convention]** `internal/ui` has EXACT-count tests that break when you add a UI row/hint: `help_test.go` `TestGetHelpSections` asserts per-section row counts (e.g. `ActionsHas11Rows` → bump to 12 when adding a help row), and `footer_test.go` `TestFooterHintSlices/GlobalHintsCount` asserts `len(globalFooterHints)`. Adding a help row in `help.go` or a hint in `footer.go` requires updating the matching count assertion (and any `…IncludesXxx` sub-test) in the same change.

- **[gotcha]** The jcodemunch repo identity for this repo is `akimovmike/abacus`, NOT `local/abacus` — `register_edit`/search with `local/abacus` or an absolute path errors `Repository index is not loadable` / `Path separator in name`. Confirm the id via `list_repos` first. `get_file_outline` wants `file_path` (not `path`) plus the `repo` id — `path` alone errors `'repo' is a required property`, and `path`+`repo` errors `Provide exactly one of 'file_path' or 'file_paths'`.
- **[gotcha]** `.beads/issues.jsonl` is a PASSIVE export — `bd create`/`bd close`/`bd update` write only the Dolt DB, so a freshly created or closed bead is ABSENT from the jsonl (`grep -c <id> .beads/issues.jsonl` → 0). Run `bd export -o .beads/issues.jsonl` to regenerate it before staging/committing it alongside related code.
- **[fix]** After `cp`-ing a fresh build over the brew Cellar binary, running it may die with `Killed: 9` (exit 137). Cause: `cp` overwrites in place = SAME inode, so macOS keeps the OLD binary's CDHash cached for that vnode and the new content fails code-signature validation. Fix: re-sign ad-hoc after the copy — `chmod u+w TARGET && codesign --force --sign - TARGET && chmod 555 TARGET`. (A new-inode atomic-swap via `mv` also sidesteps the cache.)
- **[gotcha]** `make build` without `VERSION=` stamps the binary as `version dev` (`vdev`). Pass `VERSION=x.y.z` to embed a real version string.
- **[gotcha]** Do NOT put literal `\n` sequences inside a single Bash command string (especially when wrapping in `lean-ctx -c '...'`) expecting newlines — they pass through literally and corrupt the command. Use real newlines, chain with `;`/`&&` on one line, or split into separate Bash calls.
- **[gotcha]** Serena's symbolic tools take DIFFERENT arg names and return a silent empty `{}` (no error) on no-match: `find_symbol` matches with `name_path_pattern` but returns `{}` for `name_path`, while `insert_after_symbol`/`replace_symbol_body` match with `name_path`. On an empty `{}`, retry with the other parameter name before assuming the symbol is missing.
- **[gotcha]** `tmux capture-pane`: use `-e` (lowercase) to KEEP ANSI escape sequences (needed to assert truecolor backgrounds, e.g. `48;2;224;108;117`); `-E` is the end-line flag and TAKES AN ARGUMENT, so `-peE` errors `capture-pane: -E expects an argument`.
- **[convention]** `internal/ui` has EXACT-count tests that break when you add a UI row/hint: `help_test.go` `TestGetHelpSections` asserts per-section row counts (e.g. `ActionsHas11Rows` → bump when adding a help row), and `footer_test.go` `TestFooterHintSlices/GlobalHintsCount` asserts `len(globalFooterHints)`. - **[fix]** A multi-line `git commit -F - <<'EOF' … EOF` heredoc returned an empty result and left HEAD unchanged. Write the commit message to a scratchpad file and use `git commit -F <file>` — more reliable in this harness and sidesteps the literal-`\n` corruption trap.
- **[pattern]** To show a just-mutated per-issue field immediately AND have it survive the next refresh, re-fetch it synchronously inside the command and apply it to the matching node before `forceRefresh` — do NOT rely on the post-refresh background bulk loader, which is slow/contended under real embedded-Dolt load. `invalidateCommentCache` (background re-fetch) is only the fallback.
- **[gotcha]** `./scripts/tui-test.sh keys 'Enter'` (or `'Escape'`) sends the literal characters as runes, NOT the keypress — the overlay never applies/cancels and the feature looks broken. `keys` only works for printable runes (j/k/l/f/S/etc.); use the dedicated `enter` / `escape` subcommands for special keys.

- **[gotcha]** A Go test that persists project config (`config.SaveSort`/anything under `internal/config` that resolves via `findBeadsDir`/`localProjectConfigPath`) walks UPWARD from cwd and will MUTATE the real project `.abacus/config.yaml`. In such tests, `os.Chdir(t.TempDir())` first (save/restore cwd with `defer os.Chdir(oldWd)`) so the write lands in a throwaway dir.
- **[gotcha]** Live-testing any persistence feature via `./scripts/tui-test.sh` writes the REAL `.abacus/config.yaml` (it's gitignored — `git check-ignore` confirms — so it won't pollute `git status`, but it IS your working project config). After the test, restore the baseline: `printf 'beads:\n    backend: bd\n' > .abacus/config.yaml`.
- **[gotcha]** `make ci` (the full lint+test+integration+build pre-commit gate) runs well past the 120s foreground Bash timeout. Launch it with `run_in_background` (or a longer explicit timeout), then read/tail the task output file when the completion notification arrives — don't retry it in the foreground.
