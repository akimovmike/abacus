# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[env]** `make check` runs golangci-lint v2 (`.golangci.yml` is `version: "2"`), but the binary is not pre-installed here. Install the CI-matched version before running checks: `go install github.com/golangci/golangci-lint/v2/cmd/golangci-lint@v2.12.2` (or `brew install golangci-lint`, currently 2.12.2). Without it `make check` reports lint as failed/unavailable.
- **[gotcha]** staticcheck (via golangci-lint) fails `make check` on negated comparisons: `!(x > y)` triggers `QF1001: could apply De Morgan's law`. Write the simplified form directly — `x <= y` instead of `!(x > y)`.
- **[env]** The documented TUI harness `./scripts/tui-test.sh` requires `tmux` on PATH, which is not installed by default in this environment. Without it the script prints `tmux: command not found` then `No session running` and silently does nothing. Install with `brew install tmux` first. (Fallback when tmux is unavailable: drive `./abacus` via a Python `pty.fork()`+`os.execv` harness and read the pty output.)
- **[fix]** The caliber pre-commit hook runs `caliber refresh` + `caliber learn finalize` and can hang (leaving a `caliber learn finalize --auto --incremental` process), making a normal `git commit` appear to stall. Recover: `pgrep -fl caliber` → `kill -9 <pid>`, then commit with `git commit --no-verify`.
- **[gotcha]** In bd v1.0.5 (Dolt backend) `bd export` returns counts only, NOT issue rows. Read full issues with `bd list --json --all --limit 0`, and fetch comments with `bd show --json --include-comments`.
- **[gotcha]** `br` (beads_rust) and the sibling repos `../beads`/`../beads_rust` are absent in this environment — only `bd` (`/opt/homebrew/bin/bd`) is present. br-path conformance/integration tests `t.Skip` when br is missing, so br changes cannot be live-verified here; only bd paths can.
- **[pattern]** To live-verify TUI behavior against specific data, build a throwaway beads DB in a temp dir (`bd init`, then `bd create` / `bd update <id> --status <s>` to set varied statuses), launch `./abacus` in a tmux session pointed at it, then `tmux send-keys` + `tmux capture-pane -p | sed 's/\x1b\[[0-9;]*m//g'` to assert on the rendered header/rows. `tsc`/unit tests don't prove the rendered UI.
- **[pattern]** The on-PATH `abacus` (`/opt/homebrew/bin/abacus` → Cellar) is brew-installed and the Cellar binary is read-only (`r-xr-xr-x`); a plain `cp` over it fails (Permission denied / ETXTBSY while running). For local testing only, atomic-swap: `cp ./abacus DIR/abacus.new && chmod 555 DIR/abacus.new && mv -f DIR/abacus.new DIR/abacus`. `brew upgrade`/`brew reinstall abacus` reverts it — use the `release-abacus` skill for a real, permanent release.
- **[gotcha]** The jcodemunch repo identity for this repo is `akimovmike/abacus`, NOT `local/abacus` — `register_edit`/search with `local/abacus` errors `Repository index is not loadable`. Confirm the id via `list_repos` before calling `register_edit`.
- **[gotcha]** The jcodemunch repo identity for this repo is `akimovmike/abacus`, NOT `local/abacus` — `register_edit`/search with `local/abacus` errors `Repository index is not loadable`. Confirm the id via `list_repos` before calling `register_edit`. `get_file_outline` also wants `file_path` (not `path`) plus the `repo` id — passing `path` alone errors `'repo' is a required property`, and passing `path`+`repo` errors `Provide exactly one of 'file_path' or 'file_paths'`.
- **[gotcha]** This repo's own `.beads/` is NOT a launchable store: it holds only `issues.jsonl` plus a `metadata.json` pinned to bd 0.26.0 (`database: beads.db`), with no materialized SQLite or Dolt DB on disk. Under bd v1.0.5, `bd list --json --all --limit 0` fails reproducibly (5/5) with `Error: no beads database found`, so abacus's auto-refresh errors out and it won't launch against this directory. This IS the source of the recurring "periodically shows error about running bd list" symptom — auto-refresh polls every 10s (`DefaultAutoRefreshSeconds=10`) so a failing backend re-errors every 10s. Materialize the store from the jsonl (`bd init` / import) before running abacus on this repo.
- **[pattern]** TUI errors surface only as transient ~10s toasts and are otherwise discarded — there is no persistent error record unless you opt in. To capture the full underlying error, launch `abacus --debug` and read `~/.abacus/debug.log`. Caveats: the log is TRUNCATED on every launch and is only written when `--debug` (or config `debug: true`) is set.
- **[gotcha]** `make build` without `VERSION=` stamps the binary as `version dev` (e.g. `abacus version dev`). Pass `VERSION=x.y.z` (as the deploy/release flow does) to embed a real version string into the binary.
- **[pattern]** To quantify a fix against `~/.abacus/debug.log`, capture before/after counts: `grep -c "bd run failed" ~/.abacus/debug.log` and `grep -c "context deadline exceeded"`. Because the log truncates per launch, each `abacus --debug` run is a clean trial — relaunch, drive it, then count. This is how the comment-load storm was verified (373 → 3 failures).
- **[gotcha]** In `~/.abacus/debug.log`, a `bd run failed: ... ctxErr=context deadline exceeded: signal: killed` line means abacus's OWN context timeout killed the `bd` subprocess — NOT a genuine bd error. The `ctxErr=` field (logged by `bdDoltClient.run` under `--debug`) distinguishes a timed-out/cancelled run from a real bd failure; check it before blaming bd.
- **[gotcha]** `bd show`/`bd list` against a large real Dolt-backed store can block for many seconds when a concurrent external `bd` writer holds the embedded Dolt lock — fetches queue behind it. Background loaders must give each call its own generous timeout (not one shared batch deadline) or a slow lock will mass-kill every pending subprocess at once.
- **[gotcha]** Do NOT put literal `\n` sequences inside a single Bash command string (especially when wrapping in `lean-ctx -c '...'`) expecting newlines — they are passed through literally and corrupt the command (`cd /path\necho` becomes `cd: /pathnecho`; `sleep\ntmux` makes `sleep` parse every following token, erroring `sleep: invalid time interval`). Use real newlines, chain with `;`/`&&` on one line, or split into separate Bash tool calls.
- **[gotcha]** Serena's symbolic tools take DIFFERENT arg names and return a silent empty `{}` (no error message) on no-match: `find_symbol` matched with `name_path_pattern` but returned `{}` for `name_path`, while `insert_after_symbol`/`replace_symbol_body` matched with `name_path` but returned `{}` for `name_path_pattern`. On an empty `{}`, the symbol usually exists — retry with the other parameter name before assuming it's missing.
- **[gotcha]** `tmux capture-pane`: use `-e` (lowercase) to KEEP ANSI escape sequences (needed to assert truecolor chip backgrounds, e.g. `48;2;224;108;117` for `#e06c75`); `-E` is the end-line flag and TAKES AN ARGUMENT, so `-peE` errors `capture-pane: -E expects an argument`. Pattern: `tmux capture-pane -t SESS -p -e | grep -c '48;2;R;G;B'`.
- **[convention]** `internal/ui/help_test.go` `TestGetHelpSections` asserts EXACT per-section row counts (e.g. `ActionsHas10Rows`). Adding a keybinding row to a help section in `help.go` breaks that count test — update the matching `len(sections[N].rows)` assertion (and add an `…IncludesXxx` sub-test) in the same change.
- **[fix]** A multi-line `git commit -F - <<'EOF' … EOF` heredoc returned an empty result and left HEAD unchanged (no commit landed). Write the commit message to a file and use `git commit -F <file>` — more reliable in this harness and it sidesteps the literal-`\n` command-corruption trap.
- **[pattern]** The on-PATH `abacus` (`/opt/homebrew/bin/abacus` → Cellar) is brew-installed and the Cellar binary is read-only (`r-xr-xr-x`); a plain `cp` over it fails (Permission denied / ETXTBSY while running). For local testing only, atomic-swap: `cp ./abacus DIR/abacus.new && chmod 555 DIR/abacus.new && mv -f DIR/abacus.new DIR/abacus` (or `chmod u+w TARGET && cp SRC TARGET && chmod 555 TARGET` when not running). `brew upgrade`/`brew reinstall abacus` reverts it — use the `release-abacus` skill for a real, permanent release.
- **[gotcha]** `git pull --rebase` aborts with `cannot pull with rebase: You have unstaged changes` because `bd` continuously rewrites `.beads/interactions.jsonl` (it shows as modified almost every session). Don't `git add` it into a code commit — stash just that file around the pull: `git stash push -- .beads/interactions.jsonl` → `git pull --rebase` → `git push` → `git stash pop`, then `bd dolt push`.
- **[correction]** Label rendering for the tree labels column and the detail pane must NOT use `renderPillChip` — its nerd-font pill caps (`ue0b6`/`ue0b4`) render as tofu (□?) in terminals lacking a patched font. Use `renderLabelTag(label, hex)`: plain text on the theme background when no custom color is set (i.e. transparent), a padded colored block only when a custom color is configured. Verify by byte-counting the glyphs in a tmux capture (expect 0).
- **[pattern]** To show a just-mutated per-issue field (e.g. a freshly added comment) immediately AND have it survive the next refresh, re-fetch it synchronously inside the command and apply it to the matching node (`applyCommentsToNode`) before `forceRefresh` — do NOT rely on the post-refresh background bulk loader, which is slow/contended under real embedded-Dolt load and makes the change appear lost. `invalidateCommentCache` (background re-fetch) is only the fallback when the synchronous re-fetch fails.
